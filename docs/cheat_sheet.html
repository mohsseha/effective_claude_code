Effective Claude Code — Cheat Sheet

25 Rules · Husain Al-Mohssen · March 2026 · "You don't write code. You design agent workflows that produce code."
1CONTEXT SIZE IS THE #1 LEVER
4K tokens  ≈ 97% accuracy
128K tokens ≈ 13–81% accuracy
Give each agent the smallest sufficient context. This is THE core skill. Always ask: too much or too little?
2SUBAGENTS ARE THE PRIMARY PATTERN
GOODLeader → spawns agent → agent gets [code + ticket] only
BADLeader does everything in one long session
Subagents start fresh = small context = smarter.
3LEADER NEVER IMPLEMENTS
Leader: orchestrate, delegate, review. Agents: implement, test, merge. 
Escalation sequence:
"Start an agent"
→ "Don't do this yourself"
→ "ALL work by agents, not you"
→ "Dude, have an agent do it"
4CROSS-AUDIT BEATS SELF-AUDIT
WEAKAgent checks own work (self-consistent = blind)
OKSeparate auditor agent
BESTBuilder A reviews Builder C (knows the code)
5FORCE UNDERSTANDING FIRST
"Ask me 3 yes/no questions that show you understood me."
"Draw me a diagram of how you'll do this."
"Give me 2 options with trade-offs."
The diagram IS the verification. Don't skip.
6TESTABILITY = AUTONOMY
Clear pass/fail test→ run overnight, hands off
No test→ supervise everything
Build test infrastructure FIRST. Capability = ceiling. Tests = floor.
7SPECS = REAL SOURCE CODE
Code = build artifact (disposable). Specs = source of truth (maintained). 
Multiple small specs > one big spec
One spec per agent
Out-of-date specs → broken pipeline
8AGENTS LIE
Agent: "Done! All features implemented."
Reality: Skipped hard parts. Deferred to "future versions."

ALWAYS: "Show me file + line numbers for X."
THEN:    Open the file. Read it yourself.
9CONTEXT POISONING
SymptomCorrected it, but old assumption bleeds through
CauseSelf-consistency locks wrong path
FixStart fresh. Don't fight it.
10THE COP TECHNIQUE
"There are 3 bugs in this module. Find them."
  → (even if made up — forces deeper search)

"Top 3 worst bugs? Justify ranking."
  → (forces prioritization, not just listing)
11SELENIUM IS SHOCKINGLY GOOD
1 prompt → 30 min → ~6 months manual QA equivalent
Recipe: stories/specs → Selenium → run app
        → failures → parallel fixers → re-run gate
12INDEPENDENT VERIFICATION PATHS
Path 1: Spec → Code → Unit tests (code knows code)
Path 2: Spec → Stories → Selenium  (never sees code)

Paths disagree → real bug found.
Story agents must NEVER see the code.
13OUTPUT FORMAT = YOUR REVIEW SPEED
FASTEST → SLOWEST for you to scan:
Diagrams → Tables → Bullets → Paragraphs
Tables force prioritization. Always ask for tables over prose.
14FREEZE SPECS BEFORE BUILDING
git tag -a spec-v1
Agents edit specs without asking. 5/6 modules violated frozen spec. Commit constantly. Watch diffs.
15ASK FOR ALTERNATIVES
"Give me 3 ways to do this with trade-offs."
Prevents first-instinct bias. Reveals understanding. Spends tokens thinking before coding. Can have another agent choose.
16CONTRADICTION-FINDERS ON SPECS
"Start agents with different perspectives.
 Read the specs. Highlight all contradictions."
Do before implementation. Finds issues invisible to humans.
17PERSONAS FOR AUDITING
"Agent from perspective of [foreman]."
"Agents with different themes review code."
Diverse perspectives > identical auditors.
18META-PROGRAMMING PROGRESSION
Day 1: "Write a function that..."
Day 3: "3 agents implement, 2 auditors review"
Day 5: "12-agent 5-phase pipeline w/ gates"
Natural evolution. ~44% still hands-on. Leverage = orchestration.
19COMPRESS AND RESTART
Degrading session →
  1. Ask for summary
  2. Start fresh
  3. Feed summary in
claude resume costs nothing. Don't preserve sessions out of sentiment.
20TRAINING DATA VERSION BIAS
You: "Use Claude 4.6"   It uses: 4.5  (10–20%)
Be explicit. Repeat constraint. Verify version strings.
21PLAN MODE IS UNRELIABLE
You: "Plan first, don't implement."
It:   Plans, then immediately implements.
Better: "Give me 3 alternatives" — forces thinking without building.
22SETTINGS FILE = QUALITY OF LIFE
~/.claude/settings.json
→ auto-allow: read-only, linters, test runners
Not YOLO. Curated low-risk permissions. Kills yes/yes/yes fatigue.
23YOLO MODE RULES
claude --dangerously-skip-permissions
ONLYAutomated tests + git branch + expendable machine
NEVERmain, uncommitted work, open-ended tasks
Always: commit first, git diff after.
24CLAUDE CODE >> GEMINI FOR CODING
ClaudeBest coding model + harness
GeminiGood critique & design. Weak harness, ignores "don't edit"
Use both for different strengths.
25KNOW WHEN TO HOLD 'EM
✅ Trust🔍 Review🚨 Rewrite
Boilerplate, scaffoldingBusiness logic, APIsSecurity-critical
Tests pass + spec matchEdge cases, novel algoUntestable code
Can't verify everything. Ship imperfect. That's the trade-off.
★WORKFLOW PATTERNS — QUICK REF
A · Implement
Freeze spec → tickets → agent/ticket
→ cross-audit (A reviews C) → fix → merge

B · Verify
Spec → stories (no code!) → Selenium
→ run app → fixers → re-run gate

C · Research
Researcher → Auditor (15% errors caught)
→ Summarizer. 3x slower, much safer.

D · Design
"3 ways w/ trade-offs" → choose → build
★6 TAKEAWAYS
1Naive AI makes devs slower — be intentional
2Design agent workflows, not code
3Smaller context = smarter model
4Testability determines success
5Diagram = verification
6Know when to trust vs. review
